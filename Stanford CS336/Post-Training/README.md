# Post-Training
- 预训练让语言模型具备基础能力，但不一定让它按我们想要的方式输出；因此需要收集“理想行为”的数据，并研究如何利用这些数据进一步训练和对齐模型。

Q1：如何让语言模型的回答更符合我们的要求，例如更有帮助、更安全、更遵循指令、更少胡说、更符合人类偏好？

A：**后训练，Post-Training。** 由于预训练的数据有限且信息杂乱无偏好，我们需要更针对性地用专门数据集进行进一步训练。

Q2：那么，能不能收集“我们想要的行为”来继续训练模型？

A：当然可以！这一章我们重点介绍**各种后训练方法。** 先来思考几个问题：

- 这种“我们想要的行为数据”长什么样？

  --可能是用户指令和理想回答，也可能是多个回答之间的偏好比较，或者安全规范、拒答示例、对话记录等。
  
- 拿到这些数据以后，怎么最有效地训练模型？

  --后训练的环节如何编排。是直接监督微调，还是训练奖励模型，再用强化学习优化，或者用其他对齐方法。

- 这种数据是不是也需要很大规模？

  --也就是说：少量高质量示例够不够？还是必须收集海量的人类反馈和偏好数据？

  ---

对于模型的后训练，**数据比算法更重要**，用的算法都一样，但关键在于**数据标注**，这对控制语言模型输出非常重要。

但 ChatGPT 之后，**各家公司越来越把这些数据和标注流程当作商业机密**；所以现在公开资料比早期论文少得多，想研究“真正有效的后训练配方”会更困难。

- GPT以前的早期研究论文经常把数据收集、标注标准、训练方法讲得比较清楚：

```
1. Stiennon 2020，RLHF paper

这是早期关于 RLHF 的论文之一，里面会给出比较细的标注指南，比如让标注员如何判断一个回答好不好、什么样的摘要更符合人类偏好等。

2. Bai 2022，Anthropic HH

这里的 HH 通常指 Helpful and Harmless，也就是“有帮助且无害”。Anthropic 的这类论文会更详细地说明他们如何做安全标注、如何收集偏好数据、如何让模型更安全。
```

- GPT以后，现在开源模型虽然会发布模型权重和技术报告，但很多时候会说得比较笼统。比如它们可能用了很多 distillation，蒸馏。

```
蒸馏：用一个更强模型的输出作为训练数据，去训练一个较小或较弱的模型。

例如，让强模型生成高质量回答、推理过程、偏好样本，然后拿这些数据训练自己的模型。

但问题是，很多开源发布说明不会非常详细地公开：

用了哪些老师模型；
生成了多少数据；
怎么筛选数据；
标注标准是什么；
哪些数据用于安全，哪些用于能力；
训练比例和混合方式是什么。

所以看起来开源了模型，但真正的数据配方不一定透明。
```
